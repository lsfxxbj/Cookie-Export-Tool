## Installation Guide

### Installation Steps

1. Open `chrome://extensions/` in Chrome browser
2. Enable "Developer mode" in the upper right corner
3. Click "Load unpacked"
4. Select the project folder
5. After successful installation, the extension icon will appear in the browser toolbar

### Permission Information

The browser will prompt for the following permissions during installation:

- `cookies`: Access browser cookies
- `activeTab`: Get information about the current active tab
- `storage`: Save user settings (such as language preferences)
- `<all_urls>`: Get cookies from all websites

These permissions are only used to implement extension functions and do not collect or transmit any user data.

## Basic Usage

Click the extension icon in the browser toolbar to open the popup interface.

### Interface Introduction

The extension interface is divided into two tabs:

1. **Export Tab**: For exporting Cookies
2. **Import Tab**: For importing Cookies

### Language Switching

You can select a language at the top of the interface:
- 中文
- English
- Français

## Exporting Cookies

### Export Current Website Cookies

1. Visit the target website
2. Open the extension popup
3. Select the desired export format in the export tab
4. Click the "Export Current Site Cookies" button
5. Cookies will be downloaded as a file

### Export All Cookies

1. Open the extension popup
2. Select the desired export format in the export tab
3. Click the "Export All Cookies" button
4. Cookies will be downloaded as a file

### Copy Cookies to Clipboard

1. Open the extension popup
2. Select the desired export format in the export tab
3. Click the "Copy Current Site Cookies" or "Copy All Cookies" button
4. Cookie content will be copied to the clipboard

### Export Formats

The following four export formats are supported:

1. **JSON**: JavaScript Object Notation, easy for program processing
2. **CSV**: Comma-Separated Values, suitable for spreadsheet software
3. **XML**: Extensible Markup Language, well-structured
4. **Netscape Cookie**: Netscape Cookie format, compatible with tools like curl

## Importing Cookies

### Import via File Selection

1. Open the extension popup
2. Switch to the import tab
3. Click the "Browse Files" button to select a Cookie file
4. The file will be automatically imported

### Import via Drag and Drop

1. Open the extension popup
2. Switch to the import tab
3. Drag and drop the Cookie file to the designated area
4. The file will be automatically imported

### Supported Import Formats

The following four Cookie file formats are supported for import:

1. **JSON**
2. **CSV**
3. **XML**
4. **Netscape Cookie**

## Format Specifications

### JSON Format

```json
[
  {
    "name": "cookie_name",
    "value": "cookie_value",
    "domain": ".example.com",
    "path": "/",
    "expirationDate": 1234567890,
    "httpOnly": false,
    "secure": true,
    "session": false,
    "sameSite": "lax"
  }
]
```

### CSV Format

```
domain,flag,path,secure,expiration,name,value
".example.com",FALSE,"/",TRUE,1234567890,"cookie_name","cookie_value"
```

### XML Format

```xml
<?xml version="1.0" encoding="UTF-8"?>
<cookies>
  <cookie>
    <domain>.example.com</domain>
    <flag>FALSE</flag>
    <path>/</path>
    <secure>TRUE</secure>
    <expiration>1234567890</expiration>
    <name>cookie_name</name>
    <value>cookie_value</value>
  </cookie>
</cookies>
```

### Netscape Cookie Format

```
# Netscape HTTP Cookie File
# http://curl.haxx.se/rfc/cookie_spec.html
# This file was generated by Cookie Export Tool

.example.com	FALSE	/	TRUE	1234567890	cookie_name	cookie_value
```

## Filtering Features

### Filter by Cookie Type

You can select from the "Cookie Type" dropdown menu:

1. **All**: Export all Cookies
2. **HTTPS**: Export only secure Cookies (secure)
3. **HTTP**: Export only non-secure Cookies

## Frequently Asked Questions

### 1. Why is the exported Cookie file empty?

Possible reasons:
- The current website has no Cookies
- Filter conditions are too strict, no matching Cookies
- The website has set the HttpOnly flag, which cannot be accessed via JavaScript

Solutions:
- Confirm the website actually has Cookies
- Check filter conditions
- Try visiting other websites

### 2. Why does importing Cookies show a format error?

Possible reasons:
- Incorrect file format
- Corrupted file content
- File encoding issues

Solutions:
- Confirm the file was exported from this tool or conforms to the specified format
- Check if the file is complete
- Ensure the file uses UTF-8 encoding

### 3. Why does copying Cookies to the clipboard fail?

Possible reasons:
- Browser security policy restrictions
- System clipboard permission issues

Solutions:
- Ensure copying is attempted after user interaction (such as clicking a button)
- Check if the browser allows websites to access the clipboard

### 4. How to update the extension?

In development mode:
1. Replace project files
2. Click the "Reload" button for the extension on the `chrome://extensions/` page

Store version:
- Chrome will automatically update the extension

### 5. Does the extension collect my data?

No. This extension does not collect, store, or transmit any user data. All operations are performed locally.

### 6. Are exported Cookies secure?

Exported Cookie files contain sensitive information and should be properly secured to prevent disclosure to others.

### 7. How to uninstall the extension?

1. Open `chrome://extensions/` in Chrome browser
2. Find the Cookie Exporter extension
3. Click the "Remove" button

### 8. Which browsers are supported?

Currently only Google Chrome browser is supported, requiring version 88 or higher.
